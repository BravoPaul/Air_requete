项目简介：
这是一个全球航空查询系统，你可以通过多种方式来查询航线，可以查询国家到国家的航线，城市到城市的航线，某个城市到某个国家的航线，或者某个机场到某个城市，国家的航线。然后还可以将航线以地球3D的方式更加直观的显示出来

项目实现：
首先项目包含3个文本文件，这三个文本文件可以从http://openflights.org/data.html下载，分别是airPort.dat，airLine.dat 和 route.dat 。所以项目的关键其实就是对这三个大的文本文件的处理。所以需要在项目中构建良好的数据结构，否则查询速度非常的慢。例如，为了加快查询速度，我的解决方案是，在项目中使用了hashMap<string,airport>，hashMap<string,airLine>来处理数据用airport和airline的名字来填充前面string键，后边是airport和airline的对象，因为项目中会大量的运用airport和airline的名字，而其他部分用的比较少，所以这样会很大的加快查询速度。项目中还有很多小的细节用来提升查询速度。例如我尽量将每一个查询定义一个可查询的arraylist或者hashmap，尽量避免级联查询。当查询完毕后，3D查询模块我用的是Jmonkey引擎。

项目结果：
项目基本完成了需求，只是在输入国家和城市名自动提示的时候会有问题，因为我上传的这个不是最终版本，我把最终版本弄丢了，如果有需要，我可以改一下现在的版本。3D呈现做的比较丑，不过是没有问题的。
